# Spec — v1-positioning-rebuild

## Scope
Create a usable V1 business website that aligns site messaging, page structure, case proof, and contact flow with Nebula Infinity’s approved positioning.

## In Scope
- Home page content / section restructuring
- Services index restructuring
- 3 service detail pages:
  - AI Workflow
  - AI Application
  - Web3.0 / Blockchain
- Projects / cases restructuring
- About page repositioning
- Contact page copy + inquiry taxonomy update
- Contact function real delivery path
- Japanese / English message alignment

## Out of Scope
- Blog / news
- CMS
- New authentication systems
- Separate methodology page in V1

## Information Architecture Rules
### Home must include
- Positioning hero
- 3 service lines
- Why know-how assetization matters
- Representative proof/cases
- Implementation process / trust section
- CTA

### Services index must include
- 3 service cards
- who each service is for
- representative proof mapping

### AI Workflow detail must explain
- people-dependent workflows
- know-how assetization
- workflow examples as examples only
- suitable workflow categories beyond the current examples

### AI Application detail must explain
- AI-native products / tools / systems
- proof through Japan Life Navi and Rigel

### Web3 detail must explain
- business-oriented Web3 implementation capability
- Carina as a proof sample, not the only story

### Projects / Cases must support
- workflow cases
- AI application cases
- Web3 case(s)

## Messaging Rules
- Never frame AI Workflow as only a software-development pipeline service
- Never frame workflow examples as the complete service boundary
- Never present Web3 as speculative-first or crypto-first
- Always preserve the organization-asset value proposition

## Acceptance Criteria
- [ ] Site-wide taxonomy is exactly 3 services
- [ ] Home / Services / About / Projects / Contact use the same core messaging
- [ ] Workflow value proposition explicitly includes preserving know-how as organization-owned assets
- [ ] Japan Life Navi and Rigel are visible as AI Application proof
- [ ] Web3 capability is visible even if only one public sample is shown
- [ ] Contact inquiry types align with the 3 services
- [ ] Contact submission reaches a real destination
- [ ] No blog/news route or content work is introduced into V1
